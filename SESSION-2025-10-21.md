## Technical Achievements

### H.264 Encoder Fix (Breakthrough)

**Problem**: Encoder buffered all frames, produced 0 bytes output

**Debugging Process**:
1. Created debug_encoder.rs - isolated the encoder
2. Created debug_encoder2.rs - tested with multiple frames + flush
3. Created debug_encoder3.rs - tested zerolatency config

**Solution Found**:
```rust
// x264 options for zero-latency streaming
preset=ultrafast
tune=zerolatency
intra-refresh=1
rc-lookahead=0

// Plus proper frame allocation
av_frame_get_buffer(frame.as_mut_ptr(), 0)
```

**Result**: Every frame produces output immediately!

### RGB Color Support (swscale)

**Before**: Manual grayscale YUV (U/V = 128)
**After**: Full RGB color via swscale bilinear conversion

**Encoder** (RGBA → YUV420P):
```rust
let mut scaler = ScaleContext::get(
    Pixel::RGBA, width, height,
    Pixel::YUV420P, width, height,
    Flags::BILINEAR,
)?;
scaler.run(&rgba_src_frame, &mut yuv_frame)?;
```

**Decoder** (YUV420P → RGBA):
```rust
let mut scaler = ScaleContext::get(
    Pixel::YUV420P, width, height,
    Pixel::RGBA, width, height,
    Flags::BILINEAR,
)?;
scaler.run(&decoded_yuv, &mut rgba_frame)?;
```

**Result**: Full color round-trip working perfectly!

## Key Learnings

### Decomposition Works

Following CLAUDE.md principles:
1. **Prove pipeline first** (uncompressed streaming)
2. **Debug systematically** (3 debug tools, methodical)
3. **Fix in stages** (encoder → decoder → color)
4. **Validate each step** (tests + live demo)

**Result**: Complex problem solved efficiently

### swscale is Powerful

- Replaces ~30 lines of manual YUV conversion
- Handles all edge cases (color space, chroma subsampling)
- Bilinear interpolation for quality
- Bidirectional (RGBA↔YUV420P)

### ffmpeg-next API

**Critical insights**:
- Must call `av_frame_get_buffer()` before `data_mut()`
- x264 needs `tune=zerolatency` for streaming
- `rc-lookahead=0` prevents buffering
- GOP settings affect latency significantly
